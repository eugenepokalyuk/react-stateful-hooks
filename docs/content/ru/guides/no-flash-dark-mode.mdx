# Тёмная тема без вспышки

Классическая проблема: сервер рендерит одну тему, затем клиент читает
сохранённый выбор и переключает — и страница **мигает** неправильной темой при
первой отрисовке. Этот гайд собирает три кусочка библиотеки, чтобы сделать
правильно:

1. **Блокирующий скрипт** ([`getColorSchemeScript`](#1-применяем-тему-до-отрисовки)),
   который выставляет тему на `<html>` _до_ первой отрисовки — именно он убирает вспышку
2. **Сохранённый выбор** ([`useLocalStorageState`](/ru/hooks/use-local-storage-state)
   или [`useCookieState`](/ru/hooks/use-cookie-state)), который пишет переключатель
3. **Системный фолбэк** ([`usePrefersColorScheme`](/ru/hooks/use-prefers-color-scheme))
   для тех, кто выбрал «system»

## 1. Применяем тему до отрисовки

`getColorSchemeScript()` возвращает крошечный самодостаточный сниппет.
Отрендерите его **блокирующим** inline-скриптом в `<head>`, чтобы он выполнился
до отрисовки. Он читает сохранённый выбор, откатывается на системную тему и
выставляет класс `light`/`dark` (и `color-scheme`) на `<html>`

```tsx
// Next.js App Router — app/layout.tsx
import { getColorSchemeScript } from 'react-stateful-hooks';

export default function RootLayout({ children }) {
  return (
    <html lang="ru" suppressHydrationWarning>
      <head>
        <script dangerouslySetInnerHTML={{ __html: getColorSchemeScript() }} />
      </head>

      <body>{children}</body>
    </html>
  );
}
```

Опции:

```ts
getColorSchemeScript({
  key?: string;        // ключ хранилища, по умолчанию 'theme'
  storage?: 'localStorage' | 'cookie'; // по умолчанию 'localStorage'
  attribute?: string;  // 'class' (по умолчанию) или, например, 'data-theme'
  defaultScheme?: 'light' | 'dark' | 'system'; // по умолчанию 'system'
});
```

> Поставьте `suppressHydrationWarning` на `<html>`, потому что скрипт меняет его
> до гидрации React

## 2. Переключатель

Переключатель просто читает и пишет тот же ключ. Используйте **те же `key`,
`storage` и `attribute`**, что передали в `getColorSchemeScript`

```tsx
'use client';

import { useEffect } from 'react';
import {
  useLocalStorageState,
  usePrefersColorScheme,
} from 'react-stateful-hooks';

type Choice = 'light' | 'dark' | 'system';

export function ThemeToggle() {
  const [choice, setChoice] = useLocalStorageState<Choice>('theme', 'system');
  const system = usePrefersColorScheme();
  const scheme = choice === 'system' ? system : choice;

  // Держим <html> в синхроне после первой отрисовки (её обработал скрипт).
  useEffect(() => {
    const el = document.documentElement;
    el.classList.remove('light', 'dark');
    el.classList.add(scheme);
    el.style.colorScheme = scheme;
  }, [scheme]);

  return (
    <select value={choice} onChange={(e) => setChoice(e.target.value as Choice)}>
      <option value="light">Светлая</option>
      <option value="dark">Тёмная</option>
      <option value="system">Системная</option>
    </select>
  );
}
```

## 3. Стилизация

С дефолтным `attribute: 'class'` на `<html>` появляется класс `light` или
`dark` — ровно то, что ждёт class-based тёмная тема в Tailwind:

```js
// tailwind.config.js
export default { darkMode: 'class' };
```

```css
/* или чистый CSS */
:root {
  --bg: white;
}
.dark {
  --bg: #111;
}
```

## Cookie вместо localStorage

Хотите, чтобы выбор читался ещё и на **сервере** (чтобы рендерить правильную
тему при SSR без клиентского JS)? Храните его в cookie и укажите это и скрипту,
и переключателю:

```tsx
// в <head>
getColorSchemeScript({ storage: 'cookie' });

// в переключателе
const [choice, setChoice] = useCookieState<Choice>('theme', 'system', {
  maxAge: 60 * 60 * 24 * 365,
});
```

Cookie отправляются с каждым запросом, поэтому `theme` можно прочитать на сервере
и выставить класс прямо при SSR — см. [`useCookieState`](/ru/hooks/use-cookie-state)
